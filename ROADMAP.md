# LUMIS — Roadmap

> 10-month public development plan, across 6 phases. Updated as the project evolves.

---

## Current status

**Phase 0 — Foundation** (in progress)

---

## Phase 0 — Foundation · Month 1

Establish the conceptual, visual, and technical foundations.

- [x] Complete concept document (v2, 18 chapters)
- [x] Full design system (Editorial Dark — colors, typography, components)
- [x] 4 interactive mockups built with Claude Design
- [x] Public GitHub repository
- [ ] Claude Code CLI environment with MCP servers
- [ ] Next.js 15 project scaffold
- [ ] Design system translated into Tailwind config
- [ ] CI/CD pipeline to Vercel

**Milestone**: development environment ready, first commits of real code.

---

## Phase 1 — MVP · Months 2–3

A working search engine, live online, answering real queries.

- [ ] Search UI (based on homepage mockup)
- [ ] Brave Search API integration for web retrieval
- [ ] Claude API integration for synthesis
- [ ] First answer with cited sources (Answer Page mockup brought to life)
- [ ] Basic Trust Score
- [ ] Public MVP deployed on Vercel with real domain
- [ ] Rate limiting, basic auth (Clerk)

**Milestone**: LUMIS online, answering real user queries.

---

## Phase 2 — Conversation · Months 4–5

Multi-turn conversation, memory within session, Constitutional Filter.

- [ ] Multi-turn conversation with context retention
- [ ] Basic Constitutional Filter (the 10 principles applied as a filter)
- [ ] Enhanced citation system
- [ ] Prompt engineering optimization
- [ ] Follow-up questions with full context preservation
- [ ] UI refinements based on real user feedback

**Milestone**: LUMIS feels like a conversation, not a search.

---

## Phase 3 — Differentiating features · Months 6–7

The features that make LUMIS unique in the world.

- [ ] **Temporal Lens** — time-based search and evolution timelines
- [ ] **Perspective Engine** — multi-viewpoint on contested topics
- [ ] **Knowledge Graph** — visual interactive map of concepts
- [ ] **Deep Mode** — extensive multi-source research mode
- [ ] Full **Anti-Hallucination Layer**
- [ ] Cross-Language Search foundation

**Milestone**: LUMIS offers experiences no competitor has.

---

## Phase 4 — Polish & advanced features · Months 8–9

The features that complete the vision and deepen the product.

- [ ] **True Privacy Mode** — zero-log encrypted mode
- [ ] **Research Memory** — cross-session research continuity (opt-in)
- [ ] **Synthesis Report** — session → downloadable structured document
- [ ] **Source Challenge** — challenge-and-recompute mechanism
- [ ] **Research Trails** — shareable thinking paths
- [ ] **Mistake Log** — public error admission system
- [ ] **Collaborative Research Rooms** — team-based shared context

**Milestone**: LUMIS is a complete product.

---

## Phase 5 — Presentation & outreach · Month 10

Make LUMIS visible to those who can help make it real.

- [ ] Extensive testing and bug fixing
- [ ] Beta user feedback collection and iteration
- [ ] **First public Transparency Report** — monthly cadence begins
- [ ] Pitch deck
- [ ] Outreach to Anthropic (Build in Public track, Hacker News, direct contacts)
- [ ] Hackathon participation if available

**Milestone**: LUMIS is ready to be seen.

---

## Beyond Month 10

If LUMIS earns traction and attention:

- LUMIS Education Edition (free for schools)
- LUMIS API for developers
- LUMIS Teams & Enterprise
- Scaling partnership discussions
- Infrastructure donors and funding conversations

---

## Guiding principle

**Quality over speed. Depth over features. Integrity over growth.**

LUMIS is built slowly, deliberately, and in public. Each milestone is a real release, not an announcement. Each feature is shipped when it works, not when it's promised.
