# Contributing to LUMIS

Thank you for your interest in LUMIS. This project is being built in public by a single developer, but the ideas belong to everyone who believes the way we access knowledge can be better.

## How to contribute right now

At this early stage (Phase 0–1), the best ways to contribute are:

### 💡 Share ideas and feedback
Open an [Issue](https://github.com/Teopolve/lumis/issues) with:
- Feature ideas
- Feedback on the mockups
- Critiques of the Concept Document
- Edge cases we haven't considered
- Sources to include / exclude in future Trust Score design

### 🐦 Follow the journey
The progress of LUMIS is shared publicly on X (Twitter): [@Teopolve](https://x.com/Teopolve).

### ⭐ Star this repository
It helps visibility and keeps motivation high during the long build.

---

## When will code contributions be welcome?

Starting from **Phase 2** (around month 4–5), when the MVP is stable and the codebase is ready for collaboration. Before that, the code is in rapid flux and opening it to contributors would be premature.

When that time comes, we'll open a proper contribution workflow: issue templates, PR conventions, code style, testing requirements.

---

## Principles for contributors

If you do contribute, please honor the LUMIS Constitution:

1. Truth before all else
2. Multi-perspective always
3. Admit uncertainty
4. Zero bias
5. Child protection
6. Privacy as fundamental right
7. Algorithmic transparency
8. No emotional manipulation
9. Universal accessibility
10. Public accountability

Any contribution that violates these principles — however technically correct — will be declined.

---

## Code of conduct

Be thoughtful. Be rigorous. Disagree clearly. No harassment, no bad faith, no tribalism.

LUMIS is a thinking tool. Its community should think well.

*Thank you for caring about how humanity finds truth.*
